# Equipo cliente sin respuesta (Wireless)

## Objetivo

Verificar si la falla está en la alimentación, el router, la antena o el cableado antes de derivar el caso.

---

# Paso 1. Identificar el problema

Preguntar al cliente:

- ¿El servicio no funciona desde cuándo?
- ¿Qué luces tiene encendidas el router o el adaptador?
- ¿La falla es total o intermitente?

Registrar la información.

---

# Paso 2. Identificar el tipo de conexión

Determinar cómo está alimentada la antena.

Puede ser:

- **Data + Power (Inyector PoE o adaptador).**
- **PoE Out del router (Puerto 5).**

Si no es posible identificarlo:

1. Solicitar una foto del router.
2. Solicitar una foto del adaptador (si tiene).
3. Solicitar una foto del cableado.

---

# Paso 3. Si utiliza Data + Power (Inyector PoE)

### 1. Revisar las luces del adaptador

Pedir al cliente que verifique si el adaptador tiene luces encendidas.

Registrar el resultado.

---

### 2. Reconectar el cable

Pedir al cliente que:

1. Desconecte el cable del puerto **PoE**.
2. Espere unos segundos.
3. Vuelva a conectarlo.

Consultar si el servicio volvió.

Registrar el resultado.

---

### 3. Cambiar de toma eléctrica

Pedir al cliente que conecte el adaptador en otro enchufe.

Consultar si el servicio volvió.

Registrar el resultado.

---

### 4. Si tiene MikroTik 951, 952 o AC Lite

Solicitar que conecte el cable de la antena directamente al **Puerto 5 (PoE Out)**.

Consultar si el servicio volvió.

Registrar el resultado.

---

# Paso 4. Si utiliza PoE Out del router

### 1. Verificar las luces del router

Preguntar si el router tiene luces encendidas.

### Si NO tiene luces:

1. Conectar el router en otra toma eléctrica.
2. Verificar si enciende.

Registrar el resultado.

---

### Si SÍ tiene luces:

1. Desconectar el cable conectado al **Puerto 5 (PoE Out)**.
2. Esperar unos segundos.
3. Volver a conectarlo.
4. Reiniciar el router desconectándolo de la corriente durante 10 segundos.

Aclarar que al reiniciar el router también se reinicia la antena.

Consultar si el servicio volvió.

Registrar el resultado.

---

# Paso 5. ¿Se solucionó?

## Sí

Confirmar con el cliente que puede navegar.

Registrar que el servicio quedó restablecido.

Finalizar el reclamo.

---

## No

Continuar con la derivación.

---

# Paso 6. Verificar cobertura de Fibra Óptica

Antes de derivar:

1. Verificar si el domicilio tiene cobertura de Fibra Óptica.

Si tiene cobertura:

- Generar o vincular la gestión AFO.
- Relacionar el reclamo con la gestión AFO.
- Solicitar prioridad a Coordinación.

---

# Paso 7. Derivar a Redes

Derivar cuando ocurra alguna de estas situaciones:

- El equipo no enciende.
- La antena no responde.
- El cableado o el adaptador continúan sin funcionar.
- El servicio no se recupera luego de todas las pruebas.
- Se requiere una visita técnica.

Indicar en la derivación todas las pruebas realizadas.

---

# Paso 8. Registrar la información

Antes de cerrar la atención, registrar:

- Diagnóstico.
- Tipo de conexión.
- Estado de las luces.
- Pruebas realizadas.
- Resultado de cada prueba.
- Fotos recibidas (si corresponde).
- Motivo de la derivación.
- Área a la que se deriva.
- Próximo paso informado al cliente.

---

# Resumen rápido

1. Identificar el problema.
2. Identificar el tipo de conexión.
3. Revisar luces del router o adaptador.
4. Reconectar cables.
5. Probar otra toma eléctrica.
6. Reiniciar el router o la antena.
7. Verificar cobertura de Fibra Óptica.
8. Si no funciona, derivar a Redes.
9. Registrar todas las pruebas realizadas.
